Numeric input examples

type="tel" with pattern and maxlength




Pros

  • Brings up numeric keypad on Android
  • Supports pattern and maxlength
  • Widely supported (except for Opera Mini)

Cons

  • Not semantic for non-telephone numbers

Note: Allow users not to add a phone number

type="number"

Add date:










Pros

  • Semantic
  • Brings up numeric keyboard on Android
  • In Chrome and Android, non-numeric characters can't be typed in (with exception of e, +, -, .)

Cons

  • Pressing up/down keys increases/decreases number
  • Only supports floating point numbers so credit cards (with spaces) not supported
  • Does not support commas
  • Use carefully! Do you really want to user from typing in certain characters (in Chrome/Android)?

type="text" with pattern and maxlength




Pros

  • Supports maxlength (and pattern)
  • Up/down keys don’t increase/decrease number.
  • No semantic meaning

Cons

  • Doesn’t bring up numeric keypad on Android

This kit lets you rapidly create HTML prototypes of GOV.UK services.

It contains all you need to prototype anything from simple static pages to complex, data-driven transactions.

About this page

We recommend you add links to your prototype to this page, rather than delete it. You'll find this page in the kit at '/app/views/index.html'.

You can change the service name by editing the file '/app/config.js'.
